Detailed Description

Implementation of the AND node for dHTTs.

The AND node has no constraint on the ordering of the children and so it just runs all of them in decreasing order of activation potential. The AND node also has to run all of it's children.

auction behavior for the AND node.

An AND node runs all of it's children in an unspecified order. The auction activates all children and collects their requests. Then, the request with the highest activation potential is chosen and sent up the tree. All other children are returned to the WAITING state. When the AND node is first activated it saves the resources that were passed and always passes those to the children. When each child is finished the AND adds the passed resources to a full list and then when it finishes passes the entire list of resources from the children.
