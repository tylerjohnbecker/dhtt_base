Detailed Description

sub server for a subtree in the dHTT

This class is intended to be used by GOiTRs or other similar constructs in order to communicate with the main server and give direct changes to the subtree. A parent GOiTR for instance will use this class as an interface with which it can replan it's subtree.

◆ build_subtree()

bool dhtt::SubServer::build_subtree ( )

builds the subtree that serves as the base for this action

Each goitr represents some high level action. The main idea is for each member of the tree to be left high level until the first activation. At that point the goitr should construct its own subtree and then activation should continue. For now this will be constant and the subtree will just be loaded in from a file, however, it might be better to consider this more dynamically and already have done some planning here.

Returns
True if server communication is established, false otherwise.
