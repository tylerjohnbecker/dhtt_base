Detailed Description

Base class for the plugs which provides the necessary multithreading capabilities.

This class facilitates the communication between nodes by giving a general interface of methods they can call. For now, all calls are asynchronous and the synchronous calls are obfuscated from the user.

bool dhtt::BranchPlugType::async_send_activate ( activation_goal  to_send)
inline

activate the child node attached to this plug's socket

The general method for getting responses should always be async_activate -> block_for_response -> get response. This will ensure that the state is managed correctly. If performed incorrectly some of these methods will fail. This essentially just starts the send_activate function below in another thread and keeps track of it through a thread_ptr and a future.

general initializer for the plug type

This class is meant to be created for each child and facilitates all communication from parent to child by connecting to its corresponding socket type. This class should contain all necessary systems for communication with the socket.
