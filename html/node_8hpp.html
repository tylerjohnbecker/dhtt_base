logger functions for dhtt nodes

Because dhtt nodes do not have their own logger (they are not rclcpp::Node's) this serves as a wrapper function which allows them to use the stream syntax and utilize the communication aggregator's logger with their own name appended.

Parameters
severityset by calling function to DEBUG, INFO, WARN, ERROR, or FATAL
com_ptrptr to the communication aggregator which the node class is attached to
outstream information to log

implementation of logger

DHTT_LOG should not be used, and instead one of DHTT_LOG_DEBUG, DHTT_LOG_INFO, etc. should be used by the Node class and the NodeType plugins. Each macro wraps DHTT_LOG with a given severity so that the user does not need to use that parameter when logging.
