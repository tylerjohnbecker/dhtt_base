Detailed Description

base class for the branch socket

Realistically this class is only relevant for the plug because it is intended to call the public methods of the node class and otherwise isn't necessary to be anywhere else. Added two base methods for posterity I guess in case we want heterogeneous plug/socket pairs. This facilitates communication for the activation pipeline and the maitnenance pipeline.

Note: I think I could make this more generalized to facilitate one of each communication pipeline, however, I'm not sure that this is the best way forward for the class representation.
