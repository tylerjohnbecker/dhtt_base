\doxysubsection{Detailed Description}
d\+HTT \mbox{\hyperlink{classdhtt_1_1Node}{Node}} class for all nodes on the tree 

Main \mbox{\hyperlink{classdhtt_1_1Node}{Node}} class for behaviors and tasks on the tree. Facilitates auction behavior, main tree communication protocals, status updates and maintenance, and communication with the Main Server. All node specific logic is kept as a private plugin member that is loaded when the object is created (\mbox{\hyperlink{classdhtt_1_1NodeType}{dhtt\+::\+Node\+Type}}).

\item if the node is working it runs the work\+\_\+callback of the \mbox{\hyperlink{classdhtt_1_1NodeType}{Node\+Type}} plugin and then releases and passes the appropriate resources

maintains the finite state machine status of the node after activation is received 

The activation mechanism of a node is described extensively in the paper for d\+HTT\textquotesingle{}s, however, it essentially does\+:
\begin{DoxyItemize}
\item if the node is waiting it becomes active
\item if the node is active and gets a successful activation it begins working
\item if the node is active and gets a unsuccessful activation it goes back to waiting
\item if the node is done it just immediately sends a response and stops
\end{DoxyItemize}

if the node becomes active or working it starts a separate thread and runs the activate function. if it goes back to waiting it propogates the unsuccessful activation down to it\textquotesingle{}s children as well.
